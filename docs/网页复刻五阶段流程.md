# 网页复刻五阶段流程

本文档参考 `ppt-master` 的串行流水线思想，将网页复刻流程固化为五个阶段。每个阶段必须以前一阶段产物为输入；遇到阻断条件时停止推进，不允许主观猜测真实页面。

核心流水线：

```text
复刻请求 -> 项目初始化 -> 真实素材采集 -> 复刻策略确认 -> 实现评估迭代
```

## 全局执行规则

1. 串行执行：五个阶段按顺序推进，前一阶段未完成不得进入下一阶段。
2. 门禁优先：每阶段必须产出明确文件或记录，缺失则不得继续。
3. 阻断即暂停：真实网站无法访问、出现验证码/安全验证、截图缺失、页面状态无法确认时，立即暂停并请求用户介入。
4. 证据留痕：每个页面状态都要保存真实截图、触发步骤、DOM/样式摘要或用户提供截图来源。
5. 先规格后实现：用户确认复刻方案前，不开始写页面代码。
6. 只评估三项：功能一致性、交互一致性、视觉一致性。

## 阶段一：复刻请求解析

目标：把用户的自然语言请求整理成可确认的复刻任务单。

### 输入

用户至少需要提供以下信息之一：

- 网站地址。
- 网站截图。
- 网站地址 + 截图的混合素材。

同时需要明确或由我补全以下范围：

- 要复刻哪些页面。
- 每个页面有哪些状态。
- 哪些功能需要复刻。
- 哪些功能明确不复刻。

### 处理动作

1. 判断素材来源类型：
   - `url`：由我访问真实网站并采集截图、DOM 和交互状态。
   - `screenshot`：基于用户截图复刻，不能推断截图之外的页面状态。
   - `mixed`：优先使用真实网站采集，用户截图作为补充证据。
2. 拆分页面范围：
   - 首页。
   - 列表页 / 搜索结果页。
   - 详情页 / 登录页 / 表单页。
   - 弹窗、错误页、空状态、分页页等特殊状态。
3. 拆分功能范围：
   - 输入、点击、提交、分页、筛选、Tab 切换、表单校验等。
   - 真实接口、真实登录、真实支付、真实上传等默认列为不做范围，除非用户明确要求。
4. 明确桌面端视口：
   - 默认只做 PC 端。
   - 默认采集视口为 `1365 x 768`，可按目标网站调整。

### 输出

创建复刻任务单：

```text
docs/clone-requests/{target-id}-request.md
```

任务单必须包含：

```markdown
# {网站名}复刻任务单

## 1. 素材来源

| 字段 | 内容 |
| --- | --- |
| 来源类型 | URL / 截图 / 混合 |
| URL |  |
| 用户截图 |  |
| 采集方式 | 自动采集 / 人工验证后采集 / 用户截图 |

## 2. 页面范围

| 页面 | URL / 截图 | 是否必做 | 说明 |
| --- | --- | --- | --- |

## 3. 状态范围

| 页面 | 状态 | 触发方式 | 是否必需 | 需要真实截图 |
| --- | --- | --- | --- | --- |

## 4. 功能范围

| 功能 | 是否复刻 | 复刻方式 | 说明 |
| --- | --- | --- | --- |

## 5. 不做范围

- 

## 6. 评估目标

| 指标 | 目标分 |
| --- | ---: |
| 功能一致性 | >= 90 |
| 交互一致性 | >= 85 |
| 视觉一致性 | >= 80 |
| 总分 | >= 85 |

## 7. 阻断规则

- URL 无法访问时暂停。
- 出现验证码、安全验证、登录墙时暂停。
- 必需状态截图缺失时暂停。
- 无法确认真实页面时暂停。
```

### 阶段门禁

- 任务单已创建。
- 页面范围、状态范围、功能范围、不做范围已明确。
- 用户确认任务单，或明确授权我按推荐范围继续。

## 阶段二：复刻项目初始化

目标：为该目标网站创建固定目录、评估配置草稿和素材落盘位置。

### 输入

- 已确认的复刻任务单。
- 目标网站名称和 `target-id`。

### 处理动作

1. 创建文档和日志目录：

```text
docs/clone-requests/
docs/clone-specs/
docs/ai-logs/
docs/prompts/
```

2. 创建或预留页面实现目录：

```text
src/pages/{TargetReplica}/
```

3. 创建评估配置草稿：

```text
evaluator/targets/{target-id}.json
```

4. 创建素材和基线目录：

```text
reports/baselines/{target-id}/
reports/latest/assets/{target-id}/
```

### 输出

本阶段应形成以下文件或占位：

```text
docs/clone-requests/{target-id}-request.md
docs/clone-specs/{target-id}-spec.md
docs/ai-logs/{target-id}.md
docs/prompts/{target-id}-replica.md
evaluator/targets/{target-id}.json
```

### 阶段门禁

- 目录结构存在。
- 评估目标 ID 唯一。
- 页面路由命名确定。
- 后续真实截图和评估报告有明确落盘位置。

## 阶段三：真实素材采集与状态基线

目标：在写代码前获取所有必需真实页面状态，形成复刻和评估的唯一事实来源。

### 输入

- 已确认的任务单。
- 网站 URL 或用户截图。
- 页面状态清单。

### 处理动作

#### URL 来源

1. 使用浏览器打开真实网站。
2. 逐个状态执行触发步骤。
3. 对每个状态采集：
   - 页面顶部截图。
   - 页面主体截图。
   - 页面底部截图。
   - 完整视口截图。
   - 关键 DOM 结构。
   - 关键元素样式。
   - 可点击 / 可输入元素清单。
4. 如果页面需要滚动，必须覆盖页面头部、中部和底部，不能只截首屏。
5. 如果出现验证码、安全验证、登录墙或访问异常，立即暂停，请用户介入。

#### 截图来源

1. 整理用户提供的截图。
2. 标注每张截图对应的页面和状态。
3. 检查截图是否覆盖：
   - 页面头部。
   - 页面主体。
   - 页面底部。
   - 弹窗、错误态、空状态等特殊状态。
4. 若关键状态截图缺失，向用户列出缺失项并暂停。

### 输出

每个状态都应落盘：

```text
reports/baselines/{target-id}/{state-id}/original-desktop.png
reports/baselines/{target-id}/{state-id}/original-top.png
reports/baselines/{target-id}/{state-id}/original-middle.png
reports/baselines/{target-id}/{state-id}/original-bottom.png
reports/baselines/{target-id}/{state-id}/original-dom.json
reports/baselines/{target-id}/{state-id}/capture-notes.md
```

`capture-notes.md` 记录：

```markdown
# {state-id} 真实状态采集记录

## 触发步骤

1. 
2. 

## 截图文件

- 顶部：
- 中部：
- 底部：
- 视口：

## 关键元素

| 元素 | selector / 描述 | 作用 |
| --- | --- | --- |

## 采集可信性

- 状态是否确认正确：
- 是否经过人工验证：
- 是否使用截图降级：
- 异常说明：
```

### 阶段门禁

- 每个必需状态都有真实截图。
- 页面头部、中部、底部均有素材覆盖。
- URL 来源页面已通过可信性门禁。
- 截图来源页面已明确页面和状态归属。
- 缺失状态必须补采，不能猜测。

## 阶段四：复刻策略与规格确认

目标：基于真实素材生成复刻方案，并让用户确认后锁定实现规格。

### 输入

- 复刻任务单。
- 所有真实状态截图。
- DOM / 样式 / 交互采集记录。

### 处理动作

输出一组“复刻确认项”，等待用户确认：

| 确认项 | 内容 |
| --- | --- |
| 页面范围 | 本次实际复刻哪些页面 |
| 状态范围 | 每个页面必须覆盖哪些状态 |
| 功能范围 | 哪些交互必须可运行 |
| 不做范围 | 哪些真实能力只做视觉或本地模拟 |
| 视觉重点 | 头部、主体、底部、弹窗、列表、表单等还原重点 |
| 组件拆分 | 页面将拆成哪些组件 |
| 路由设计 | 本地访问路径 |
| 评估目标 | 功能、交互、视觉阈值 |

用户确认后，写入规格文件：

```text
docs/clone-specs/{target-id}-spec.md
```

规格文件必须包含：

```markdown
# {网站名}复刻规格

## 1. 页面与状态设计

| 页面 | 状态 | 设计说明 | 对应真实截图 |
| --- | --- | --- | --- |

## 2. 组件设计

| 组件 | 负责区域 | 状态变化 | 参考截图 |
| --- | --- | --- | --- |

## 3. 功能设计

| 功能 | 触发方式 | 复刻行为 | 是否调用真实接口 |
| --- | --- | --- | --- |

## 4. 视觉设计

| 区域 | 还原要点 |
| --- | --- |
| 页面头部 |  |
| 页面主体 |  |
| 页面底部 |  |
| 弹窗/浮层 |  |

## 5. 评估配置设计

| 状态 | selector | 交互步骤 | 截图基准 |
| --- | --- | --- | --- |
```

同时更新机器可读配置：

```text
evaluator/targets/{target-id}.json
```

### 阶段门禁

- 用户已确认复刻策略。
- 每个状态都有设计方案和真实截图映射。
- 功能、不做范围、评估目标均已锁定。
- 评估配置能覆盖所有必需状态。

## 阶段五：实现、评估与迭代交付

目标：根据锁定规格实现复刻页面，并通过评估报告迭代到目标分。

### 输入

- 已确认的复刻规格。
- 真实素材基线。
- 评估配置。

### 处理动作

1. 先实现页面骨架和路由。
2. 按状态逐个实现：
   - 初始态。
   - 输入态。
   - 结果态。
   - 分页态。
   - 空状态 / 错误态。
   - 弹窗 / 浮层。
3. 每完成一个状态，运行评估：
   - 功能一致性。
   - 交互一致性。
   - 视觉一致性。
4. 对低分项按报告建议修复：
   - 功能低分：补齐缺失状态或关键功能。
   - 交互低分：修复点击、输入、提交、分页、状态切换。
   - 视觉低分：查看 diff 图，修复布局、尺寸、字体、颜色、间距。
5. 保留 AI Prompt 和人工修改记录。

### 输出

```text
src/pages/{TargetReplica}/
evaluator/targets/{target-id}.json
reports/latest/report.md
reports/latest/index.html
docs/ai-logs/{target-id}.md
docs/prompts/{target-id}-replica.md
```

### 阶段门禁

- 本地页面可访问。
- 所有必需状态可触发。
- 评估报告只包含功能、交互、视觉三项。
- 报告标明评估方式：实时原站采集评估或截图/DOM 基线降级评估。
- 总分达到任务单目标，或明确记录未达标原因和下一步修复建议。

## 五阶段产物总览

| 阶段 | 关键产物 | 是否阻断 |
| --- | --- | --- |
| 阶段一：复刻请求解析 | `clone-requests/{target-id}-request.md` | 是 |
| 阶段二：复刻项目初始化 | 目录结构、评估配置草稿、日志文件 | 是 |
| 阶段三：真实素材采集与状态基线 | 每个状态的真实截图、DOM、采集记录 | 是 |
| 阶段四：复刻策略与规格确认 | `clone-specs/{target-id}-spec.md`、评估配置 | 是 |
| 阶段五：实现、评估与迭代交付 | 复刻页面、评估报告、Prompt 记录 | 是 |

## 与 `ppt-master` 的对应关系

| `ppt-master` 思路 | 网页复刻对应阶段 |
| --- | --- |
| Source Content Processing | 阶段一：复刻请求解析 |
| Project Initialization | 阶段二：复刻项目初始化 |
| Strategist Confirmations | 阶段四：复刻策略与规格确认 |
| Image / Resource Acquisition | 阶段三：真实素材采集与状态基线 |
| Executor + Quality Check + Export | 阶段五：实现、评估与迭代交付 |
