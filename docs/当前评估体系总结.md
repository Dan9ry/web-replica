# 当前评估体系总结

更新时间：2026-05-30

本文总结当前 `web-replica` 项目的网页复刻一致性评估体系。评估目标不是判断原站本身是否优秀，而是判断复刻页面是否在用户指定范围内，尽可能一致地还原原站的桌面端视觉、结构、内容、功能与交互。

## 核心原则

1. 先有可信原站基线，再做评分。
2. 评估阶段不重新交互式采集原站，只使用 Phase 3 已确认的截图、DOM 摘要、关键 selector 和文本基线。
3. 原站基线缺失、错误页、验证码页、安全验证页、空白页或关键状态缺失时，评估必须中断或标记硬门禁失败，不允许主观猜测。
4. 最终产品必须是可交互网页，不能用整页截图、大区域截图、canvas 截图或图片热区伪装。
5. 复刻范围不等于功能范围：视觉必需区域仍要出现并参与视觉评分，但用户未要求的功能不进入交互用例。

## 评估输入

每个复刻 project 使用自己的配置文件：

```bash
EVAL_TARGET_CONFIG=projects/{target-id}/config/target.json npm run eval
```

评估器读取：

- `projects/{target-id}/config/target.json`
- `projects/{target-id}/baselines/{state-id}/original-dom.json`
- `projects/{target-id}/baselines/{state-id}/original-{viewport}.png`
- 当前复刻页面 URL，例如 `http://127.0.0.1:5173/replica/{target-id}`

评估只针对当前指定 project，不提供“评估所有项目”的默认选项。

## 硬门禁

硬门禁优先于评分。硬门禁失败时报告仍可生成，但结论为 `failed`，分数只用于定位问题，不作为通过依据。

| 门禁 | 通过条件 |
| --- | --- |
| 原站基线可信 | required 状态都有可读取的 DOM 摘要、截图、可信 URL/title、核心文本和关键 selector |
| 复刻页可运行 | 本地服务和目标路由可访问，页面不是空白页 |
| required 状态可达 | 每个 required state 都能被采集 |
| 禁止截图伪页面 | 主体 UI 不能由整页截图、大图背景、canvas 或图片热区伪装 |
| 核心交互无阻断错误 | 核心流程无 uncaught exception、白屏、死循环 |
| 评估可复现 | 视口、配置、基线、截图、报告产物完整 |

## 六维总分

总分采用 100 分制，只评估桌面端：

```text
总分 = 功能正确性       * 0.25
     + 交互流程一致性   * 0.20
     + 视觉一致性       * 0.25
     + 结构语义一致性   * 0.10
     + 内容数据一致性   * 0.10
     + 工程可维护性     * 0.10
```

## 默认验收阈值

当前 skill 默认验收阈值：

```text
total score >= 85
functionality >= 85
interaction >= 85
visual >= 60
structure >= 80
content >= 80
engineering >= 80
```

若用户要求更严格的阈值，使用更严格值。除非用户明确接受更低目标，否则不主动降低这些默认阈值。

## 六维指标说明

| 维度 | 权重 | 评估重点 |
| --- | ---: | --- |
| 功能正确性 | 25% | required 状态覆盖、核心用例、输出结果、边界/异常态 |
| 交互流程一致性 | 20% | 点击、输入、Enter、focus/blur、分页、反馈和状态切换 |
| 视觉一致性 | 25% | 整页截图、区域 diff、SSIM、布局、样式、字体、颜色、间距 |
| 结构语义一致性 | 10% | 用户复刻范围内真实 DOM、控件语义、列表/表单/导航结构 |
| 内容数据一致性 | 10% | 可见文案、错误提示、结果字段、分页数字、状态文案 |
| 工程可维护性 | 10% | 可运行、可构建、project-local 源码组织、报告证据完整 |

## 视觉一致性

视觉一致性判断页面“看起来像不像”。顶部、底部、账号入口、应用入口、辅助链接等可见区域即使没有功能，也会影响视觉分。

每个状态生成：

- `original-{viewport}.png`
- `replica-{viewport}.png`
- `diff-{viewport}.png`

没有配置 `regions` 时：

```text
视觉分 =
  整页视觉分 * 75%
+ 布局分     * 15%
+ 样式分     * 10%
```

整页视觉分：

```text
整页视觉分 = 像素差异分 * 55% + SSIM 分 * 45%
像素差异分 = 100 - diffRatio * 100
SSIM 分 = ssim * 100
```

配置了 `regions` 时，会更重视关键区域：

```text
视觉分 =
  区域视觉分 * 45%
+ 整页视觉分 * 15%
+ 布局分     * 20%
+ 样式分     * 15%
+ 5 分补正
```

区域视觉分：

```text
区域视觉分 =
  pixelScore * 45%
+ ssimScore  * 35%
+ bboxScore  * 12%
+ styleScore * 8%
```

## 结构语义一致性

结构语义一致性现在按用户复刻意图加权，不要求原站完整 DOM 逐项复刻。

当前结构分：

```text
结构语义分 =
  范围内 selector 覆盖 * 80%
+ 大类结构相似度       * 10%
+ 关键控件覆盖         * 4%
+ 可聚焦/可操作性      * 3%
+ DOM 文本真实性       * 3%
```

核心配置是 `structureSelectors`：

```json
{
  "selector": ".result-link",
  "purpose": "content",
  "expectedCount": 8,
  "weight": 4
}
```

`purpose` 分层：

| purpose | 含义 | 权重倾向 |
| --- | --- | --- |
| `functional` | 用户要求真实交互的控件，如搜索框、登录输入框、按钮 | 高 |
| `content` | 用户要求展示的数据/文案，如搜索结果、错误提示 | 较高 |
| `structural` | 页面骨架，如表单、列表、分页容器 | 中 |
| `visual` | 视觉必须出现但无需功能，如顶部账号区、应用入口、页脚链接 | 低 |

例如用户要求 8 条结果，实际实现 5 条，则该 selector 的覆盖分约为：

```text
5 / 8 = 62.5
```

但页脚链接这类 `visual` 区域只用低权重参与结构评分；它们应出现在页面中，但不要求实现点击后的真实功能。

## 功能与交互

功能正确性主要来自：

- required 状态覆盖率。
- 核心用例断言通过率。
- 输出结果正确率。
- 边界/异常态通过率。

交互流程一致性来自 `target.json` 中的 `interactions`。评估器会用 Playwright 执行动作和断言，例如：

- `fill`
- `click`
- `press`
- `hover`
- `focus`
- `expectVisible`
- `expectTextIncludes`
- `expectValue`
- `expectUrlIncludes`

用户未要求的功能不应加入交互用例。例如顶部账号菜单、应用入口、页脚链接如果只是视觉复刻，就不需要验证点击后的功能。

## 内容数据一致性

内容数据一致性检查用户可见信息是否对齐，包括：

- 页面核心文案。
- 输入框 placeholder。
- 错误、空态、loading 文案。
- 结果列表标题、摘要、字段。
- 分页数字、统计文案。

支付、登录、账号、安全验证类页面只采集可见 UI 证据，不 dump 隐藏 input、cookie、localStorage、sessionStorage 或 token 字段。

## 工程可维护性

工程可维护性检查：

- 复刻页面可运行。
- 源码位于 `projects/{target-id}/page/`。
- 组件、数据、状态、样式拆分清晰。
- 不使用截图当网页。
- 评估产物和日志完整。

评估输出写入当前 project：

projects/{target-id}/evaluation/latest/

报告结构：

8. 证据中心。

## 迭代规则

如果评估未达阈值：

1. 按报告中的低分项和问题列表修复。
2. 只重新评估当前 project。
3. 每轮记录修改内容、运行命令、分数变化和剩余问题。
4. 最多进行 3 轮修复与重新评估。
5. 即使分数未达标，也不得用截图页替代可交互网页。

## 当前边界

- 当前不单独评估性能、响应式、SEO 或完整 WCAG 合规。
- 当前主要面向桌面端页面。
- 历史对比入口已保留，但历史趋势分析仍可继续增强。
- 视觉分受字体、浏览器渲染、动态内容和第三方页面差异影响较大，因此默认阈值低于功能和交互阈值。
