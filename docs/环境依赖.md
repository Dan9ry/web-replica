# 环境依赖

本文档记录项目运行、开发、测试和一致性评估所需的环境依赖。

## 基础环境

| 依赖 | 建议版本 | 说明 |
| --- | --- | --- |
| Node.js | 22.x LTS | 项目 TypeScript 与 Node 类型按 Node 22 配置 |
| npm | 随 Node.js 安装 | 用于安装依赖和运行脚本 |
| Chromium | 由 Playwright 安装 | 用于 E2E 测试、截图采集和评估流程 |

推荐先确认本机版本：

```bash
node -v
npm -v
```

## 首次安装

```bash
npm install
npx playwright install chromium
```

说明：

- `npm install` 安装 `package.json` 中声明的项目依赖。
- `npx playwright install chromium` 安装 Playwright 测试和评估需要的 Chromium 浏览器。
- 当前项目不依赖后端服务或数据库。
- 复刻页面优先使用自定义 CSS 贴近原站；`antd` 仅在后台、表单或辅助界面合适时按需使用，避免默认组件样式影响复刻还原度。

## 常用环境变量

评估器不内置固定目标，必须通过 `EVAL_TARGET_CONFIG` 指定当前复刻项目配置：

```bash
EVAL_TARGET_CONFIG=projects/{target-id}/config/target.json npm run eval
```

兼容旧命令：

```bash
EVAL_TARGET_CONFIG=projects/{target-id}/config/target.json npm run eval:interactive
```

当前 `eval:interactive` 等同于 `npm run eval`。评估阶段只使用 `projects/{target-id}/baselines/` 中已经确认的原站截图/DOM 基线，不再交互式打开原站。

评估输出写入当前 project，而不是统一主目录：

```text
projects/{target-id}/evaluation/latest/
├── summary.json
├── details.json
├── artifacts-index.json
├── report.md
├── index.html
├── captures/
└── assets/
projects/{target-id}/evaluation/history/{timestamp}/
```

Windows PowerShell 可使用：

```powershell
$env:EVAL_TARGET_CONFIG="projects/{target-id}/config/target.json"; npm run eval
```
